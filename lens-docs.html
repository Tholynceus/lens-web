LENS Documentation

Read the dev behind any token

LENS turns any X profile or Base contract into a plain-language rug-risk verdict, by reading the deployer wallet on-chain instead of trusting the bio. Bios lie, chains don't.

Overview

Open an X profile and LENS reads the bio for a wallet, a contract, or a GitHub link, then pulls the real on-chain history of whoever deployed the token and shows it right under the profile. No tab switching, no block explorers, no copy-paste.

The result is one of three calls you can act on in a second:

CLEAR  nothing alarming on-chain CAUTION  mixed signals, look closer STOP  behavior that rugs people

a bio claims one thing, the deployer wallet tells the truth

LENS reads Base chain and tokens launched through Bankrbot. Multi-chain is on the roadmap.

What you need

  • The LENS Chrome extension for scanning X profiles inline, or
  • The public API and MCP server for agents, bots and scripts

Install the extension

LENS is a Chrome MV3 extension. Once installed it runs on x.com automatically, with no configuration.

  1. Add to Chrome. Install from the Web Store, or load the unpacked build from the repo for the latest version.
  2. Open any X profile. LENS injects a panel under the bio within about a second.
  3. Read the verdict. Trust score, red lines and the dev's deploy history appear inline, no clicks needed.

Permissions, in plain terms

PermissionWhat it is actually for
x.com accessRead the bio on the profile you are viewing and inject the panel
storageRemember your settings and cache recent scans
networkCall the LENS API for on-chain data

LENS is read-only. It never asks to connect a wallet, never holds keys, and never signs anything.

Your first scan

Open the profile of any token dev on Base. The panel appears under the bio with everything LENS could read about the deployer.

the panel LENS injects under a profile

Reading the panel

  • Trust score — a 0 to 100 read on the deployer, higher is safer.
  • Verdict chip — the headline CLEAR / CAUTION / STOP.
  • Stat row — deploys, sells, linked accounts, funding source.
  • Red lines — the specific behaviors that pulled the verdict down.

The verdict system

Every scan collapses to one of three calls. The logic is worst-signal-wins: a single STOP-level red line is enough to mark the whole token STOP, no averaging that hides one fatal flaw behind nine green ticks.

the worst signal sets the call
VerdictMeaningWhat to do
CLEARNo red lines on-chainStill your call, but nothing screams rug
CAUTIONSoft signals worth a lookRead the red lines before sizing in
STOPBehavior that rugs holdersWalk away, or size as pure gamble

The seven red lines

For a token, LENS runs a transparent rule set called tokenRedLines(). Each rule maps to a flag, and the highest flag becomes the verdict. Nothing is a hidden black-box score, you can see exactly which line tripped.

Red lineFlagWhy it matters
Dev sold a prior tokenSTOPA deployer who dumped before will likely dump again
Serial deployerSTOPDozens of launches is a rug factory pattern
Dev claimed fees and leftSTOPTook the creator fees, no intent to build
Bundled supplyCAUTIONLaunch wallets hold a big share, dump risk
Fresh funding trailCAUTIONWallet funded minutes before launch from a CEX
Hidden linked accountsCAUTIONAlt identities tied to past failed launches
Thin or no historyCAUTIONNothing to verify, treat as unproven

A CLEAR verdict means no red line tripped, not a guarantee. Always do your own research before aping.

Trust score

For a dev or wallet, LENS also returns a 0 to 100 trust score, a quick read on the deployer built from the same on-chain signals. It starts high and is pulled down by each red flag in the wallet's history.

RangeBandRead
80 – 100solidClean, consistent, holds through launches
45 – 79mixedSome flags, read the detail before trusting
0 – 44riskyStrong rug-pattern history

Scan an X profile

This is the default. Open a profile and LENS reads the bio for a wallet, a contract or a GitHub link, resolves the deployer, and shows the verdict inline. You do nothing.

If a bio has no on-chain footprint, LENS says so plainly instead of inventing a score, an empty profile is information too.

Address matching is case-insensitive, so a checksummed or lowercase address in a bio both resolve the same dev.

Scan a contract

You can point LENS straight at a token contract, in the extension or through the public API. It returns the same verdict and red lines for whoever deployed it.

bash
# scan a Base token contract
curl "https://lens-liard.vercel.app/api/lookup?contract=0x9a3c…b7e2"
javascript
const res = await fetch(
  `https://lens-liard.vercel.app/api/lookup?contract=${ca}`
)
const { data } = await res.json()

if (data.found) console.log(data.verdict, data.red_lines)

Scan a wallet

Skip the token and read the dev directly. Pass a wallet address to get its deploy history, sell behavior, funding trail and trust score, the full picture behind the person.

bash
# read a deployer wallet directly
curl "https://lens-liard.vercel.app/api/lookup?wallet=0x8618…C2BE"

You can also scan by X handle with ?username=, which resolves the bio to a wallet first, then runs the same checks.

B20 tokens

B20 is Base's native token standard. Unlike a normal ERC-20, it bakes issuer powers straight into the token, freeze-and-seize, unlimited mint, pause, and transfer policies. LENS reads every one of those powers on-chain and turns them into a verdict.

a B20 scan reads issuer powers, not just balances

B20 is not active on Base mainnet yet. LENS is already built to read it, so the day Base switches it on, scanning works through the same endpoint. See the B20 page.

What LENS reads

The scanner asks the B20 factory whether a contract is a B20, then reads its state directly off the precompile, read-only, no key, no signature.

Dev claim & fees

On Bankrbot launches, the creator can claim accrued trading fees. LENS shows whether the dev has claimed, how much, and whether they kept building afterward or vanished.

A dev who claims fees and goes silent is a classic soft rug. That pattern alone trips a STOP red line.

Serial deployer

LENS counts every token a wallet has deployed and how each one ended. A handful of honest launches is normal; dozens of dead tokens is a rug factory.

The deploy count and the fate of past tokens (sold, abandoned, still live) feed directly into the trust score.

PleaseBro detection

"PleaseBro" tokens are launched through a known farm pattern that mass-produces copy tokens to bait buyers. LENS flags wallets and contracts that match this fingerprint.

When a deployer shows the PleaseBro signature, the verdict reflects it before you read a single reply in the thread.

CA & bio history

Devs swap the contract address in their bio between launches. LENS keeps a history of which CAs a profile has promoted over time, so a "first ever token" claim is easy to disprove.

A bio that has cycled through five contracts in a month is its own warning.

Bundled wallets

At launch, insiders often spread supply across many wallets in a single block to hide concentration. LENS detects bundled buys and estimates how much of the supply launch wallets actually control.

High bundled supply means a coordinated dump can hit at any moment, a CAUTION line.

AI verdict

On top of the rule-based red lines, LENS can run an AI pass that reads the full on-chain picture and writes a short, plain-language summary of why a dev looks safe or sketchy.

The rules decide the verdict; the AI explains it in a sentence you can paste into a group chat.

Token health

Beyond the dev, LENS reads the token itself, liquidity, holder spread, and whether the LP is locked or sitting in a wallet that can pull it.

A great dev on a token with unlocked LP is still a risk, and LENS keeps the two readings separate so you see both.

Linked accounts

Devs run alt accounts to recycle a reputation after a failed launch. LENS surfaces X accounts tied to the same wallet or funding source, so a "new" builder with a buried history is exposed.

Funding trail

LENS traces where the deployer wallet got its gas. A wallet funded minutes before launch from a fresh CEX withdrawal is a throwaway, very different from a wallet with months of real history.

A fresh funding trail is a CAUTION line on its own.

Smart followers

LENS labels the on-chain-relevant accounts that follow a profile, traders, deployers, known names, so you can tell organic traction from a bot-padded follower count. Each gets a short role label generated from its own footprint.

GitHub intel

If a bio links a GitHub, LENS reads the public profile, repos, age and activity, and shows whether the "open source" claim holds up or the account was created last week with one empty repo.

Community tags

Holders can tag wallets and profiles (for example scammer, builder, known team). Tags are community-voted, so a single bad actor cannot brigade a label, and they appear alongside the on-chain verdict.

API reference

The same data behind the extension is available over a public HTTP endpoint. One route, three ways to ask.

endpoint
GET https://lens-liard.vercel.app/api/lookup
ParamExampleResolves
username?username=lnsx_ioBio → deployer wallet → verdict
contract?contract=0x9a3c…Token → deployer → verdict
wallet?wallet=0x8618…Wallet history → trust score

All reads are case-insensitive and need no API key. Be reasonable with request volume.

MCP & agents

LENS ships an MCP server so any agent can run a rug check mid-conversation, the same verdict you see, available as a tool call. Connect it in one line:

bash
claude mcp add --transport http lens https://lens-liard.vercel.app/api/mcp
an agent calls LENS like any other tool

The server exposes three tools, scan an X username, a contract, or a wallet, each returning the verdict and red lines as structured data. Full guide on the Agents page.

Response schema

Every lookup returns the same shape. Fields that don't apply are omitted rather than null.

FieldTypeMeaning

data.red_linesarray{ flag, label, detail }
data.deployer_walletstringResolved deployer address
data.sourcestring"b20" for B20 reads

FAQ

Does LENS touch my wallet?
No. It is read-only, never connects a wallet and never signs.

Which chain?
Base, with tokens launched through Bankrbot. More chains are planned.

A token shows "not found", is that bad?
It means LENS has no on-chain record for it yet, common for brand-new or non-indexed tokens. Treat unknown as unproven, not safe.

Is a CLEAR verdict a guarantee?
No. It means no red line tripped. Always do your own research.
